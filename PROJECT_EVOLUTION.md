# 项目演进计划 - Project Evolution Plan

> "每2.5个小时推进这个项目的演进和发布"
> — Chao

## 🎯 总体目标

按照质量保障体系和发布流程，持续、稳定地推进 Vibe Todo 项目的演进，确保：
- ✅ 质量第一 - 测试覆盖和质量门禁
- ✅ 小步快跑 - 每次迭代小而可控
- ✅ 可追溯 - 清晰的变更记录和版本管理
- ✅ 自动化 - 尽量自动化流程，减少人工操作

## ✅ 单次演进任务步骤

每 2.5 小时执行的任务：

1. **检查状态** - 读取演进状态文件
2. **检查 token** - 确保 token 充足（如不足则暂停）
3. **执行当前任务** - 按阶段计划执行下一个任务
4. **运行测试** - 执行质量门禁检查
5. **更新状态** - 保存当前进度
6. **汇报进展** - 向用户汇报本次任务结果

---

## 🛡️ 质量检查（每次必须）

- [ ] 所有现有测试通过
- [ ] 代码检查通过 (`ruff check`)
- [ ] 没有破坏现有功能
- [ ] 如果有新代码，有对应的测试
- [ ] 文档已更新（如需要）

---

## 📊 Token 检查

- 在任务开始前检查 token 使用情况
- 如果 token 不足（例如剩余 < 10k），暂停任务并报告
- 保留当前状态，等待下次继续

---

## 🎯 发布标准

只有在以下情况才发布新版本：
1. 一个阶段的所有任务完成
2. 所有测试通过
3. 质量门禁 100% 满足
4. CHANGELOG 已更新
5. 文档已完善
